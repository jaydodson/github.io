环境搭建

2019-08-03 信安之路 mysql, nginx, php-fpm 评论

采用nginx+php-fpm+mysql的架构。

系统安装

用的是ubantu的系统，18.04版本。
搭建的时候使用的电脑性能不太好，就在服务器上搞了。
首先是虚拟化出一台主机，安装 Ubuntu18.04 Server

安装完成系统之后，首先安装openssh-server

sudo apt install openssh-server

接下来设置下固定IP.
这里需要注意，ubuntu从17.10开始，已放弃在/etc/network/interfaces里固定IP的配置，即使配置也不会生效，而是改成netplan方式 ，配置写在/etc/netplan/*.yaml文件里，直接执行命令sudo vi /etc/netplan/50-cloud-init.yaml 编辑即可，格式如下图所示。

最后还需要执行 sudo network apply 使配置生效

MySQL安装

执行 sudo apt install mysql-server 安装即可，默认版本是5.7.28

MySQL安装完之后执行mysql_secure_installation对root进行安全配置

MySQL的root账户默认是无法远程连接的，可以看一下

update user set plugin="mysql_native_password";
更改登陆验证方式为密码验证登陆

update user set authentication_string=password('password') where user='root';
更改root的密码

update user set host="%" where user="root";
允许root账户远程登陆。

select host,user,authentication_string,plugin from user;查看MySQL所有的账户以及允许登陆的host、密码的MD5值和登陆验证方式。

如果想使root账户可以远程登陆，还需要修改MySQL的配置文件。
sudo vi /etc/mysql/mysql.conf.d/mysqld.cnf
将bind-address的注释去掉并将127.0.0.1改为0.0.0.0即可

最后测试一下MySQL的更改

安装nginx 以及 PHP

sudo apt install nginx #安装nginx

sudo apt-get install php7.2 php7.2-fpm php7.2-mysql #安装PHP

这时就已经全部安装完成啦，但是如果自己写一个php文件就会发现，php其实是不可用的。
我们需要更改nginx的配置
sudo vi /etc/nginx/sites-available/default

以及php7.2-fpm的配置
sudo vim /etc/php/7.2/fpm/pool.d/www.conf

再次使用php文件测试，至此环境安装结束。

初步安全加固

  • /etc/pam.d/common-password #密码强度设置

  • /etc/shadow #在该文件中设置账号的密码使用期限等信息

用户名:密码:上次修改用户账号密码的时间:密码最小使用期限:密码最大使用期限:密码过期提醒时间:密码过期锁定账号时间:账号过期时间:保留

密码区有！是不能登录系统的

上次修改密码时间为0表示下次登录强制修改密码，空表示关闭密码过期功能

  • sudo apt-get install fail2ban 使用fail2ban用做SSH防爆破

/etc/fail2ban/jail.conf #默认配置文件，软件更新时被覆盖

/etc/fail2ban/jail.local #本地配置文件中，软件更新不被覆盖（cp conf重命名成local，local的优先级比conf高）

ignoreip=127.0.0.1/8 #无视密码验证失败

bantime=600 # 屏蔽该ip的秒数 -1 表示永久

findtime= 1 #多少秒查看一下日志

maxretry=6 #连着几次错误屏蔽

[Jail设置]

enabled = true #启用，优先级比全局高

filter=sshd

fail2ban-client status #查看jail列表

fail2ban-client status sshd

iptables -L -n查看防火墙规则

iptables -D 详细规则 #在iptables防火墙中删除规则

fail2ban-client set sshd unbanip 192.168.1.1 #在fail2ban中删除，实际上也是删除 iptables中的规则

重启服务时会去读取/var/log/auth.log 日志，在过去10分中内，达到maxretry会被禁

  • Ubuntu默认的防火墙管理工具UFW(基本当作主机防火墙使用)

    UFW默认不记录日志,需要手动进行设置。

sudo ufw logging on|off|LEVEL(low/medium/high/full)

/var/log/ufw.log 日志文件
